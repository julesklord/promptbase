# Governance - Promptbase

## Overview
This document outlines the decision-making process and authority structure for the Promptbase project. As an open-source, community-driven initiative, clear governance ensures the quality and consistency of the prompt library.

## Authority & Approval
For the current phase of the project, all technical and content changes require explicit review.

- **Lead Maintainer**: Jules Martins (@julesklord)
- **Approval Policy**: All Pull Requests (PRs) must receive an "Approve" review from **Jules Martins** before they can be merged into the `main` branch.

## Component Ownership

| Component | Content Curator | Technical Lead |
|---|---|---|
| Prompt Database (`prompts.json`) | Jules Martins / Community | Jules Martins |
| Core Architecture (`js/`, `css/`) | N/A | Jules Martins |
| Testing Infrastructure | N/A | Jules Martins |

## Becoming a Maintainer
Currently, maintainer roles are not being expanded. If you are a frequent contributor and wish to help with triage, please open an Issue to discuss.
